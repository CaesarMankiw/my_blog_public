## 0. 主题可行性判断

### 0.1 结论

这三篇文献**可以构成第二次 pre 的主题**，但建议不要继续沿用过窄的标题“隐式与连续谱优化机制”。更准确的主题应当是：

> **从显式快慢切换到隐式思考形态控制：潜空间压缩、表征速度调控与多模态交织推理**

原因如下：

第一次 pre 的 Group 1 关注的是**显式自适应决策机制**：模型或外部框架通过规则、阶段任务或奖励信号显式判断“何时快、何时慢”。这类方法通常仍然以显式 token-level CoT 为主要推理载体，快慢切换也多表现为二元路径选择、阶段切换或难度感知长度控制。

第二次 pre 的三篇文献则进入另一条路线：它们不再只问“是否进入慢思考”，而是进一步问：

1. **思考是否必须以显式文本 token 形式展开？**
2. **快慢是否只能是二元切换，还是可以是连续可控的速度谱？**
3. **推理是否只能在语言中发生，还是可以在语言、视觉和多模态交织空间中发生？**

因此，三篇论文可以形成如下递进链条：

```text
CoLaR
显式 token CoT → 压缩 latent reasoning
关注：推理载体从文本 token 转向潜空间

Controlling Thinking Speed
固定推理风格 → 表征方向连续调控
关注：推理速度从离散快慢转向连续可控

ThinkMorph
单一语言 CoT → 文本/视觉/交织推理形态切换
关注：推理模态从单一文本扩展到多模态形态空间
```

## 0.2 三篇文献的匹配度

| 文献 | 与主题匹配度 | 推荐保留程度 | 主要理由 |
|---|---:|---:|---|
| **CoLaR** | 高 | 强烈建议保留 | 典型“隐式/潜空间推理”工作，直接把 token-level CoT 压缩到 latent chain，并支持动态压缩因子 |
| **Controlling Thinking Speed** | 高 | 强烈建议保留 | 典型“连续速度控制”工作，通过表征编辑实现快慢思考的连续 steering |
| **ThinkMorph** | 中-高 | 建议保留，但需调整主题表述 | 不是纯效率优化论文，而是“推理形态/模态空间”的扩展；放在第三篇作为 Group 2 的外延扩展是合理的 |

### 关键判断

**CoLaR 和 Controlling Thinking Speed 是主题核心。**  
它们分别对应：

- **隐式推理载体**：CoLaR 将思考从显式 token 序列压缩到 latent representation。
- **连续推理速度**：Controlling Thinking Speed 通过 steering vector 在表示空间中连续调节快慢思考。

**ThinkMorph 是主题边界上的扩展。**  
它并不主要解决“token 变少”或“速度变快”的问题，而是解决“推理以什么形态发生”的问题。它适合放在第三篇，用来说明第二次 pre 不仅关注效率，还关注**思考形态从 token → latent → representation → multimodal interleaving 的扩展**。

如果汇报场景要求主题非常收敛，可以将 ThinkMorph 替换为更纯粹的 latent reasoning 或 efficient reasoning 文献，例如：

- **Coconut**：连续 latent thought 方向，与 CoLaR 更接近；
- **CODI**：通过 self-distillation 将 CoT 知识迁移到 latent reasoning；
- **TokenSkip / ThinkPrune / O1-pruner / L1 / s1**：如果希望主题转向“长 CoT 剪枝、预算控制、过度思考治理”。

但如果希望第二次 pre 与第一次 pre 形成更明显的“范式扩展”，则保留 ThinkMorph 更有展示价值。

## I. 引言：从显式快慢决策到隐式思考形态控制

第一次 pre 中的三篇工作可以概括为**显式自适应决策机制**：

- **DynaThink**：在推理时用一致性和推理复杂度作为规则，显式判断问题走 Fast 路径还是 Slow 路径；
- **Thinker**：把问答任务拆成 Fast Thinking、Verification、Slow Thinking、Summarization 四个阶段，通过 RL 内化快慢循环；
- **FAST-GRPO**：在 LVLM 的 GRPO 中引入难度估计、长度奖励和动态 KL，使模型根据问题难度调节推理深度。

这类方法的共同点是：**快慢思考的切换仍然是可观察、可命名、可解释的外显机制**。无论是规则分流、阶段任务，还是 reward shaping，它们都显式指定了“快路径”与“慢路径”的边界。

然而，这一范式存在三个限制：

1. **推理载体仍然主要是文本 token**  
   长 CoT 的成本来自自回归 token 生成。即使能够判断何时快慢，慢思考本身仍然需要生成大量可见 token。

2. **快慢边界仍然偏离散**  
   显式机制通常将问题分为 fast/slow、easy/hard 或阶段化流程，但真实推理深度更接近连续谱，而不是二元分类。

3. **推理形态仍然偏语言中心**  
   对多模态任务而言，仅靠文本 CoT 描述视觉内容可能不足。复杂视觉推理可能需要生成、编辑、聚焦或重构中间视觉状态。

因此，第二次 pre 可以将主题推进为：

> **隐式与连续谱优化机制：模型不再只显式选择“是否慢想”，而是通过 latent compression、representation steering 和 multimodal interleaving 改变思考本身的载体、速度与形态。**

## II. Think Silently, Think Fast: Dynamic Latent Compression of LLM Reasoning Chains

### 2.1 基础画像

- **标题**：Think Silently, Think Fast: Dynamic Latent Compression of LLM Reasoning Chains
- **简称**：CoLaR，Compressed Latent Reasoning
- **作者/机构**：Wenhui Tan, Jiaze Li, Jianzhong Ju, Zhenbo Luo, Ruihua Song, Jian Luan；Renmin University of China / Xiaomi
- **发表信息**：NeurIPS 2025；arXiv:2505.16552v5
- **代码**：https://github.com/xiaomi-research/colar
- **一句话快读**：CoLaR 将显式 CoT 推理链压缩为可自回归预测的 latent variables，每个 latent 表示多个 token 的语义，从而让模型在潜空间中“静默思考”，并通过动态压缩因子控制推理速度。

---

### 2.2 问题与解决方案

#### 核心痛点

CoT 和长链推理显著提升了数学推理能力，但其代价是高昂的自回归 token 生成成本。传统优化方法通常在 token 层面减少冗余，例如跳过低信息 token、提示模型生成更短推理、或提前终止推理。这些方法虽然有价值，但仍然受限于稀疏 token 序列。

CoLaR 的核心问题是：

> 是否可以让模型不再逐 token 显式展开推理，而是在更稠密的 latent space 中完成一部分思考？

#### 核心思想

CoLaR 认为推理链中的相邻 token 具有可压缩性。若将连续 $c$ 个 reasoning tokens 的 embeddings 合并为一个 compressed embedding，则模型可以用更短的 latent chain 表示原始推理链。

其核心目标有三点：

1. **压缩推理 token**  
   将显式 reasoning tokens 映射到压缩 latent embeddings。

2. **自回归预测 latent reasoning chain**  
   使用 Latent Head 预测后续 compressed embeddings，而不是只预测下一个文本 token。

3. **保留探索-利用能力**  
   Latent Head 输出概率分布而不是确定性 latent，使 RL 可以探索不同 latent reasoning paths，并偏向更短、更有效的路径。

### 2.3 方法流程

#### 阶段一：SFT 中的动态 latent compression

对于训练样本：

```text
Question → Reasoning Chain → Answer
```

设原始 reasoning chain 的 token embeddings 为：

$$
e_r = e_r^{1:L_r}
$$

CoLaR 在每个训练 step 随机采样压缩因子：

$$
c \in [1, c_{\max}]
$$

然后将连续 $c$ 个 reasoning token embeddings 合并为一个 compressed embedding：

$$
L_c = \left\lceil \frac{L_r}{c} \right\rceil
$$

论文中采用按 $\sqrt{c}$ 归一化的 embedding sum，而不是简单 mean pooling。原因是高维 token embedding 直接平均可能改变原始 embedding 分布的方差结构。

#### 阶段二：辅助压缩 token 预测

为了让模型理解 compressed embeddings，CoLaR 不只监督最终答案，还监督模型恢复压缩 reasoning tokens。

论文中的压缩 token 预测损失可概括为：

$$
\mathcal{L}_{comp}
=
-
\frac{1}{L_a + L_c}
\sum_{i=1}^{L_a+L_c}
\log p([t_c, t_a]_i \mid [e_c, e_a]_{1:i-1}, e_q)
$$

其中：

- $t_c$ 是从原 reasoning tokens 中按 group 采样得到的压缩 token 监督；
- $t_a$ 是 answer tokens；
- $e_c$ 是 compressed reasoning embeddings；
- $e_a$ 是 answer embeddings；
- $e_q$ 是 question embeddings。

这个目标使模型能够在 compressed embedding 输入下仍然维持对 reasoning chain 和 answer 的建模能力。

#### 阶段三：Latent Head 自回归推理

CoLaR 增加一个 Latent Head，用于预测下一个 compressed embedding 的分布。推理时：

```text
Question + “Let’s think c × faster”
→ Latent Head 生成 latent reasoning variables
→ Language Head 判断何时终止 latent reasoning
→ 输出最终答案
```

因此，推理过程不再完全依赖显式文本 token，而是在 latent variables 上自回归展开。

#### 阶段四：RL 增强

CoLaR 进一步使用 GRPO 进行后训练。每个问题采样一组 latent reasoning outputs，计算相对 reward，并将 reward 按 token/latent chain 长度平均。

这个设计的作用是：

- 正确但长的 latent path 得到较弱强化；
- 正确且短的 latent path 得到更强强化；
- 错误 path 被惩罚；
- 模型逐渐学会探索多样 reasoning paths，同时利用更紧凑的有效路径。

### 2.4 实验与观察

#### 数据与评价设置

论文主要在数学推理任务上评价，包括 GSM8K、GSM8K-hard、SVAMP、MultiArith，并在更困难的 MATH 任务上验证 RL 增强效果。
![[组会pre/推荐系统发展史/推荐系统发展史——快慢思考/屏幕截图 2026-05-20 162636 1.png]]
#### 主要结论!

1. **与 latent-based baselines 相比，CoLaR 在相近压缩比下取得更高准确率。**
2. **相较显式 CoT，CoLaR 可以显著减少 reasoning chain length，但通常仍存在一定性能损失。**
3. **RL 阶段表现出“先变长探索，再变短利用”的训练动态。**
4. **动态压缩因子训练比固定压缩因子训练更具泛化能力。**
5. **CoLaR 可以泛化到未见过的整数 compression factors，但对非整数或超出训练范围的 compression factor 泛化不足。**

---

### 2.5 优点

- **从根本上改变推理载体**：不是简单剪短 token，而是将推理迁移到 latent space。
- **支持动态速度控制**：通过提示 compression factor 控制推理链压缩程度。
- **适合与 RL 结合**：概率化 Latent Head 为 latent reasoning 提供探索空间。
- **与第一场 pre 的显式机制形成清晰对比**：CoLaR 不再显式判断 fast/slow，而是让推理在潜空间中以不同密度展开。

### 2.6 局限

- **不是所有任务都超过显式 CoT**：多数情况下更接近效率-性能折中，而非全面优于 CoT。
- **压缩因子仍是离散控制变量**：对非整数 $c$ 或超过训练范围的 $c$ 泛化不足。
- **latent reasoning 的可解释性弱于文本 CoT**：虽然可以通过 embedding similarity 做案例解释，但整体透明度仍有限。
- **训练和实现复杂度高于普通 CoT/SFT**：需要修改模型结构，引入 Latent Head 和额外训练目标。

---

### 2.7 铁三角分析

- **效果（Effectiveness）**  
  在数学推理任务中，CoLaR 可以在压缩推理链的同时保持相对较高准确率；RL 后训练能够进一步激活 latent reasoning 的探索能力。

- **效率（Efficiency）**  
  通过将多个 token 压缩为一个 latent variable，CoLaR 显著降低 reasoning chain length，尤其适合长 CoT 成本过高的场景。

- **成本（Cost）**  
  训练成本和系统改造成本较高，需要模型结构扩展、latent objective 设计和 RL 后训练；推理阶段虽然 token 更少，但 latent generation 仍需模型前向计算。

## III. Controlling Thinking Speed in Reasoning Models

### 3.1 基础画像

- **标题**：Controlling Thinking Speed in Reasoning Models
- **作者/机构**：Zhengkai Lin, Zhihang Fu, Ze Chen, Chao Chen, Liang Xie, Wenxiao Wang, Deng Cai, Zheng Wang, Jieping Ye；Zhejiang University / Alibaba Cloud 等
- **发表信息**：NeurIPS 2025；arXiv:2507.03704v2
- **代码**：https://github.com/D2I-ai/thinking-speed-control
- **一句话快读**：这篇论文发现部分 LRM 内部存在快慢思考的表示方向，并通过 representation editing 在推理时连续 steering 模型的 thinking speed，实现快思考省 token、慢思考提准确率，并进一步用实时难度估计实现动态速度调整。

---
### 3.2 问题与解决方案

#### 核心痛点

大型推理模型（LRM）擅长长链慢思考，但常常带来大量 token 消耗和高延迟。简单地用 prompt 要求模型“少想一点”或强制早停，往往会破坏自然推理流，导致准确率明显下降。

论文提出两个核心问题：

1. **如何控制 LRM 的 thinking speed？**
2. **什么时候应该加速或减速？**

#### 核心思想

论文不是从训练任务或奖励函数入手，而是从模型内部表征入手：

> 快思考和慢思考可能对应 LRM representation space 中的不同方向。若能找到 slow-to-fast 的 steering vector，就可以在推理时通过 hidden state intervention 连续控制模型的推理风格。
![[组会pre/推荐系统发展史/推荐系统发展史——快慢思考/屏幕截图 2026-05-20 162957.png]]
---

### 3.3 方法流程

#### 观察一：LRM 中存在快慢思考的“自然开关”

作者首先观察 DeepSeek-R1-Distill-Qwen 系列模型的长短回答，发现短回答和长回答往往具有不同的 leading words。

例如，将 `<think>` 后的开头强制为 `To`，会显著缩短模型的 reasoning tokens，同时保留部分推理能力。这说明 LRM 内部可能天然存在快慢思考模式，只是缺少稳定、连续、可控的切换机制。

#### 表征读取：构造 slow-fast direction

对同一个问题 $q_i$，收集：

- fast response：$a_i^f$
- slow response：$a_i^s$

构造刺激对：

$$
T_i^+ = (q_i, a_i^f)
$$

$$
T_i^- = (q_i, a_i^s)
$$

在指定层 $l$ 提取最后 token 的 hidden states：

$$
h_{i+}^l,\ h_{i-}^l
$$

计算差分方向，并通过 PCA 提取第一主成分作为 steering vector：

$$
v^l
$$

该向量被解释为从 slow-thinking 到 fast-thinking 的方向。

#### 表征控制：推理时 hidden state intervention

在推理生成每个 token 时，对目标层 hidden state 做如下修改：

$$
h^l \leftarrow h^l + \alpha v^l
$$

其中：

- $\alpha > 0$：增强 fast-thinking，输出更短、更直接；
- $\alpha < 0$：增强 slow-thinking，输出更长、更审慎；
- $\alpha = 0$：原始模型。

这使 thinking speed 从二元切换变为连续 steering。

---

### 3.4 自适应速度控制

![[组会pre/推荐系统发展史/推荐系统发展史——快慢思考/屏幕截图 2026-05-20 163046.png]]
固定 $\alpha$ 只能让模型整体变快或变慢，但真实推理过程应当是局部动态的：

- 简单步骤快速通过；
- 困难步骤放慢分析。

因此，论文引入实时 reasoning difficulty estimation。其基本思路是用 contrastive decoding 信号估计当前生成片段的复杂程度，并据此动态切换 steering intensity。

概念上可以理解为：

```text
低难度片段 → α > 0 → fast steering
高难度片段 → α < 0 → slow steering
```

最终形成 intra-chain 的动态快慢控制，而不是 task-level 的一次性路由。

---

### 3.5 实验与观察

#### 数据与评价设置

论文在 AIME24、MATH-500、GPQA Diamond、LiveCodeBench 等推理任务上评估，模型包括 DeepSeek-R1-Distill-Qwen-7B/32B、QwQ-32B、Qwen3-8B 等。
![[组会pre/推荐系统发展史/推荐系统发展史——快慢思考/屏幕截图 2026-05-22 221806.png]]
#### 主要结论

1. **单纯 leading word 限制即可显著减少 token，但准确率保留有限。**
2. **representation steering 比 prompt-based budget forcing 更平滑，且在相同长度下通常有更好准确率。**
3. **负向 steering 可增加推理长度并提升复杂任务准确率。**
4. **动态速度控制在平均意义上同时提升准确率并降低 token 使用。**
5. **不同 LRM 对 steering intensity 的敏感性不同，长度控制还不完全均匀。**

---

### 3.6 优点

- **无需训练或微调目标模型**：属于 inference-time plug-in。
- **连续控制快慢思考**：通过 $\alpha$ 实现速度谱，而不是 fast/slow 二分。
- **粒度细**：可以在 token-level 或 generation-level 介入 hidden states。
- **可与其他推理策略正交结合**：例如 self-consistency、Best-of-N、parallel search 等。

### 3.7 局限

- **依赖可提取的快慢样本对**：需要先从目标模型或同类模型中构造 fast/slow thought pairs。
- **不同模型的 steering 敏感性不同**：同一个 $\alpha$ 不一定对应相同 token budget。
- **难度估计与 speed adjustment 尚未完全端到端优化**：论文也指出未来可将难度估计和最优 steering intensity 更有机地结合。
- **表征方向的语义解释仍有限**：PCA 方向有效不等于完全理解了快慢思考机制。

---

### 3.8 铁三角分析

- **效果（Effectiveness）**  
  负向 steering 可提升复杂推理准确率；动态控制在多模型、多 benchmark 上呈现较好的平均效果。

- **效率（Efficiency）**  
  正向 steering 可显著压缩回答长度；动态控制能在简单片段节省 token，在困难片段保留慢思考。

- **成本（Cost）**  
  不需要重新训练模型，但需要构造 steering vector、选择介入层和调节 $\alpha$；部署上需能访问并修改模型 hidden states，因此更适合开源模型或可控推理引擎。

## IV. ThinkMorph: Emergent Properties in Multimodal Interleaved Chain-of-Thought Reasoning

### 4.1 基础画像

- **标题**：ThinkMorph: Emergent Properties in Multimodal Interleaved Chain-of-Thought Reasoning
- **作者/机构**：ICLR 2026, [Jiawei Gu](https://openreview.net/profile?id=~Jiawei_Gu4), [Yunzhuo Hao](https://openreview.net/profile?id=~Yunzhuo_Hao1), [Huichen Will Wang](https://openreview.net/profile?id=~Huichen_Will_Wang1), [Linjie Li](https://openreview.net/profile?id=~Linjie_Li1), [Michael Qizhe Shieh](https://openreview.net/profile?id=~Michael_Qizhe_Shieh1), [Yejin Choi](https://openreview.net/profile?id=~Yejin_Choi1), [Ranjay Krishna](https://openreview.net/profile?id=~Ranjay_Krishna1), [Yu Cheng](https://openreview.net/profile?id=~Yu_Cheng1)
- **一句话快读**：ThinkMorph 不是简单让模型“多写文字推理”，而是训练统一多模态模型生成文本与视觉交织的 reasoning traces，使模型在文本、视觉和多模态交织三种思考形态之间切换，并在视觉中心任务中获得更强泛化。

---
### 4.2 为什么它放在 Group 2 中是合理的

严格来说，ThinkMorph 不是典型的“推理加速”论文。它的核心贡献不是减少 token，也不是直接控制速度，而是改变推理的**模态形态**：

```text
text-only CoT
→ image-only / visual thought
→ interleaved text-vision thought
→ task-dependent mode switching
```

因此，它适合放在第二次 pre 的第三篇，用于扩展主题边界：

> 如果 CoLaR 说明“思考可以不完全写成 token”，Controlling Thinking Speed 说明“思考速度可以连续调节”，那么 ThinkMorph 说明“思考形态可以在语言、视觉和多模态交织之间切换”。

它与前两篇的共同点是：都在削弱“显式文本长链 CoT 是唯一慢思考形式”的假设。

---

### 4.3 问题与解决方案

#### 核心痛点

多模态推理不是单次视觉感知，也不是单纯把图像描述成文本后再做 CoT。许多视觉中心任务需要模型主动观察、定位、重构或修改中间视觉状态。

仅使用 text-only CoT 存在问题：

- 对空间结构、局部细节、图表区域等信息表达不足；
- 文本描述可能丢失视觉细节；
- 对 jigsaw、visual search、chart refocus 等任务，推理需要视觉操作参与。

#### 核心思想

ThinkMorph 训练统一多模态模型生成 interleaved reasoning trace：

$$
T = (\hat{m}_1, \hat{m}_2, \ldots, \hat{m}_n)
$$

其中：

$$
\hat{m}_i \in \{\hat{t}_i, \hat{v}_i\}
$$

即每一步 thought 可以是文本 token，也可以是视觉 token。视觉 thought 通过特殊 token 标记 image generation 区间。

### 4.4 方法流程

#### 数据构建

ThinkMorph 重点构造四类需要主动视觉参与的任务：

1. **Jigsaw Assembly**  
   先用文本描述各图像块，再生成视觉拼接候选，最后用文本检查视觉一致性。

2. **Spatial Navigation**  
   先用文本识别迷宫结构，再用视觉 token 渲染路径，最后用文本验证路径安全性。

3. **Visual Search**  
   先用文本定位目标区域，再生成 bounding box 聚焦局部，最后基于局部视觉证据回答。

4. **Chart Refocus**  
   先用文本确定关键图表元素，再通过视觉高亮或框选聚焦区域，最后进行数值读取与计算。

这些任务的共同点是：文本和视觉不是同义重复，而是互补推进推理。

#### 模型与训练

ThinkMorph 基于统一多模态模型，能够同时处理文本生成和图像生成。训练目标由两部分组成：

- 文本生成：next-token negative log-likelihood；
- 图像生成：对生成图像与目标图像进行 MSE 监督。

总体思想是将 interleaved thought 看作一个统一的 autoregressive sequence，并在其中联合建模文本 token 与视觉 token。

---

### 4.5 三层 morphing

ThinkMorph 的“morphing”可以分为三层：

#### 1. Token-Level Morphing

同一条 reasoning chain 内部交织文本 token 与视觉 token。

```text
text thought → visual operation → text verification → answer
```

#### 2. Chain-Level Morphing

模型可以根据任务选择：

- text-only reasoning；
- vision-only reasoning；
- interleaved reasoning。

这对应推理形态层面的自适应模式选择。

#### 3. Test-Time-Level Morphing

在 Best-of-N 或 test-time scaling 下，不同候选 chain 可能自然呈现不同 modality patterns。多模态交织扩大了搜索空间，提升了候选推理轨迹的多样性。

### 4.6 实验与观察
![[组会pre/推荐系统发展史/推荐系统发展史——快慢思考/屏幕截图 2026-05-20 163238.png]]

#### 数据与评价设置

论文在视觉中心任务和多模态泛化 benchmark 上评价，例如 visual search、jigsaw assembly、MMVP、SAT、VStar 等。

#### 主要结论

1. **Interleaved reasoning 在需要主动视觉参与的任务上有效。**
2. **模型出现未显式监督的视觉操作能力，例如 zoom-in、crop、image inpainting、multiple bounding boxes。**
3. **模型可以在 text-only、vision-only 和 interleaved reasoning 之间进行任务相关的模式选择。**
4. **在 Best-of-N test-time scaling 下，interleaved chains 因为搜索空间更大，通常比单模态 chains 有更强扩展性。**
5. **Interleaved reasoning 并非总是优于 text-only reasoning；在偏 OCR 或表格文本处理任务上，额外视觉 token 可能引入冗余。**

---

### 4.7 优点

- **突破语言中心 CoT**：将 reasoning trace 扩展到文本-视觉交织空间。
- **适合视觉中心任务**：尤其适合需要定位、重构、空间操作和细粒度视觉检查的任务。
- **出现模式切换能力**：模型不一定总是使用 interleaved reasoning，而是可能根据任务选择不同推理模式。
- **为 test-time scaling 提供更大搜索空间**：不仅可以扩大文本路径，还可以扩大视觉操作路径。

### 4.8 局限

- **主题匹配度弱于 CoLaR 和 Controlling Thinking Speed**：它不是主要解决快慢速度，而是解决推理形态与模态交织。
- **训练成本和数据构造成本高**：需要高质量 interleaved reasoning traces。
- **适用任务有条件**：主要适合 active visual engagement 明显的任务；对文本化表格或 OCR-like 任务不一定占优。
- **论文状态需谨慎**：上传文件显示为 ICLR 2026 under review，报告中不宜把其作为已正式发表工作表述。

---

### 4.9 铁三角分析

- **效果（Effectiveness）**  
  对视觉中心任务效果明显，尤其在需要局部观察、空间重构和视觉操作的任务上。

- **效率（Efficiency）**  
  不一定减少 token 或计算成本；它的效率收益更多来自“更合适的推理形态”，而不是单纯缩短输出。

- **成本（Cost）**  
  成本较高，需要统一多模态生成模型、高质量交织数据和图像生成/编辑能力；部署复杂度显著高于文本 LRM 控制方法。

## V. 三篇工作的横向对比

| 维度 | CoLaR | Controlling Thinking Speed | ThinkMorph |
|---|---|---|---|
| 核心问题 | token CoT 太长 | LRM 慢思考太冗余 | 纯文本 CoT 不适合视觉中心推理 |
| 推理载体 | compressed latent variables | hidden representations | text tokens + visual tokens |
| 控制方式 | compression factor $c$ | steering intensity $\alpha$ | modality / reasoning mode |
| 是否需要训练 | 需要 SFT + 可选 RL | 不需要训练目标模型，但需构造 steering vector | 需要 interleaved finetuning |
| 快慢形式 | latent chain 长短与压缩率 | 连续速度谱 | 文本/视觉/交织模式选择 |
| 主要收益 | 降低显式推理链长度 | 同时控制准确率-效率 trade-off | 提升视觉中心任务推理能力 |
| 主要风险 | latent 可解释性和泛化边界 | 不同模型 steering 敏感性不同 | 主题偏多模态推理，不是纯效率优化 |

## VI. 与第一次 pre 的关系

第一次 pre 可以概括为：

```text
显式路由 / 显式阶段 / 显式 reward
```

第二次 pre 可以概括为：

```text
隐式载体 / 连续表征 / 多模态形态
```

更具体地：

| 第一次 pre：显式自适应决策 | 第二次 pre：隐式与连续形态控制 |
|---|---|
| 判断任务是否走 fast/slow | 改变思考本身的表示形式 |
| 外部规则或 reward 显式定义路径 | latent / representation / modality 内部调控 |
| 多数仍在文本 token CoT 上工作 | 不再局限于显式文本 CoT |
| 关注“何时慢想” | 关注“如何以不同密度、速度和模态思考” |

因此，两次 pre 可以组成一条更完整的技术演进线：

```text
固定 CoT / Self-Consistency
→ 显式自适应快慢决策
→ 隐式 latent reasoning
→ 表征级连续 speed control
→ 多模态 reasoning mode morphing
```

## VII. 对推荐系统与 CogRec 的启发

这三篇文献对推荐场景中的 reasoning/routing 设计有直接启发，但不能机械迁移。

### 7.1 对 sid_routing / CogRec 的潜在启发

1. **CoLaR 启发：SID 推理链是否可以压缩？**  
   当前推荐 CoT 如果显式生成完整 reasoning text，可能带来额外 token 成本和训练噪声。可以思考是否存在更短的 routing latent、cluster transition embedding 或 compressed reasoning code 表达用户-物品路径。

2. **Controlling Thinking Speed 启发：routing 强度是否可连续调节？**  
   当前 fast/slow 或 RA/routing 常被设计成离散路径。表征 steering 的思想提示我们，推理深度也许可以通过连续强度控制，而不是只做二元路由。

3. **ThinkMorph 启发：推荐推理不一定只有文本形态**  
   推荐中的“认知地图”、SID 层级、HNSW 邻接结构、cluster transition 本身可以视作非语言 reasoning substrate。未来可以探索文本解释与结构化路径共同推进推荐推理。

### 7.2 需要注意的边界

- CoLaR 面向数学推理，不能直接证明 latent compression 对推荐有效；
- representation steering 需要可控 hidden states，黑盒模型场景不可用；
- ThinkMorph 面向统一多模态模型，与当前 SID 推荐的模态条件不同；
- 推荐任务的评价指标是 Hit/NDCG，而不是 reasoning correctness，因此效率-准确率 trade-off 的定义需要重新设计。

## IX. 总结

第二次 pre 的核心结论可以这样表述：

> 第一组工作解决的是“模型如何显式判断何时快想、何时慢想”；第二组工作进一步转向“思考本身如何被重新表示、压缩和调控”。CoLaR 将推理链从文本 token 压缩到 latent space，Controlling Thinking Speed 通过 representation steering 实现连续速度调节，ThinkMorph 则把推理形态扩展到文本、视觉和交织模式。三者共同说明，快慢思考不再只是二元决策问题，而是一个关于推理载体、推理速度和推理模态的连续控制问题。

## X. 汇报时建议避免的表述

1. 不要说 ThinkMorph 是“推理加速方法”。  
   更准确：它是“多模态推理形态扩展方法”。

2. 不要说 CoLaR 已经全面优于显式 CoT。  
   更准确：它在效率上显著压缩推理链，在部分设置下接近或超过 CoT，但整体仍是效率-效果折中。

3. 不要说 representation steering 完全解释了快慢思考机制。  
   更准确：它找到了可操作的表征方向，但方向的因果语义和跨模型稳定性仍需验证。

4. 不要将 ThinkMorph 作为正式发表论文表述。  
   上传文件显示其仍处于 ICLR 2026 under review 状态，建议写作“under review work”或“recent preprint/work under review”。
